# v4.0.0 — Pluggable LazyArray API

## Context

`mbo_utilities` is built around two functions: `imread()` returns a lazy array, `imwrite()` writes one. Today, both are scattered:

- **`imread()` is a 400-line if/elif chain** (`mbo_utilities/reader.py:114-511`) hard-coding every supported format and isoview sub-layout.
- **`pipeline_registry.py` exists but is never used for dispatch** — only `gui/widgets/pipelines/settings.py` reads it, and only as metadata.
- **5D is not enforced.** Audit (16 array classes):
  - Suite2pArray, ZarrArray, ScanImage subclasses report `ndim==5`.
  - TiffArray, BinArray, H5Array, NumpyArray, NWBArray report **natural rank** (drops singletons).
  - IsoviewArray returns 4D *or* 5D depending on `single_timepoint`.
  - Even the 5D classes disagree on `__getitem__`: ScanImage subclasses normalize 5D keys to 3D internally; ZarrArray strict-unpacks `(t,c,z,y,x)`; BinArray rejects 5D entirely.
- **Only TIFF classes have `can_open()`.** Every other format is dispatched by extension/marker checks inside `imread()`.
- The feature layer (`DimLabels`, `FrameRate`, `VoxelSize`, `RoiFeatureMixin`, `PhaseCorrection`) is already clean: pure duck-typed attribute presence, zero coupling. **This part is already v4-shaped — leave it alone.**

The intent of v4 is to make the API forkable: a third-party repo (e.g. an LSM-specific lab) should ship one file declaring `class MyFormatArray(LazyArray): ...`, register it, and have `imread()` pick it up — no edits to `mbo_utilities`.

User decisions (confirmed via questions):

1. **`arr.shape` is always a 5-tuple TCZYX.** `arr.ndim == 5`. No natural-rank. Hard break from v3.
2. **Both registration paths**: pyproject `entry_points` group + runtime `register_array_class()`.
3. **Dispatch via `can_open()` classmethod with `PRIORITY` ordering.** Replace the `imread()` if/elif chain.
4. **Tiny core + opt-in mixins.** Required: `_shape5d`, `dtype`, `__getitem__`, `metadata`, `can_open`, `PRIORITY`.
5. **Lazy contract**: int/slice/Ellipsis on any of 5 axes; no fancy/boolean indexing required; must not materialize axes the user didn't request.
6. **5D by default; squeeze is opt-in** via `arr.squeeze()` and `imread(path, squeeze=True)`.
7. **Migration**: hard break, `MIGRATION.md`.
8. **Complex layouts (isoview)**: one class per layout with distinct `PRIORITY`s.

## The v4 LazyArray

New module: `mbo_utilities/lazy_array.py`.

```python
class LazyArray:
    """Base class every imread-returnable array inherits from.

    Required: subclasses must implement _shape5d, __getitem__, can_open,
    and have a `dtype` and `metadata` property. PRIORITY may be overridden.
    """
    PRIORITY: ClassVar[int] = 50

    # required
    def _shape5d(self) -> tuple[int, int, int, int, int]: ...
    def __getitem__(self, key) -> np.ndarray: ...
    @classmethod
    def can_open(cls, path: Path) -> bool: ...

    # required as instance attrs/properties
    dtype: np.dtype
    metadata: dict

    # provided by base
    @property
    def shape(self) -> tuple[int, int, int, int, int]:
        return self._shape5d()
    @property
    def ndim(self) -> int:
        return 5
    @property
    def nt(self) -> int: return self._shape5d()[0]
    @property
    def nc(self) -> int: return self._shape5d()[1]
    @property
    def nz(self) -> int: return self._shape5d()[2]
    @property
    def ny(self) -> int: return self._shape5d()[3]
    @property
    def nx(self) -> int: return self._shape5d()[4]

    def __array__(self, dtype=None, copy=None) -> np.ndarray: ...  # default: arr[0,0,0]
    def squeeze(self) -> SqueezedView: ...
    def _imwrite(self, outpath, **kw): return _imwrite_base(self, outpath, **kw)
```

`Shape5DMixin` (in `arrays/_base.py:40-111`) is **folded into `LazyArray`** and removed. `nt`/`nc`/`nz`/`ny`/`nx`/`shape5d`/`source_path` move with it. `shape5d` becomes an alias for `shape` (kept for one minor's grace period, deprecated).

The `__getitem__`  is documented in the base class docstring with a worked example. Plugin authors implement `__getitem__(self, key)` where:
- `key` is normalized via `_normalize_key(key, ndim=5)` (already exists, `arrays/_base.py:114-122`).
- After normalization, `key` is a length-5 tuple of `int | slice`.
- Plugins must read only the requested ranges; they must not loop over a full axis to satisfy a single index.

`__array__` defaults to returning a single representative `(Y,X)` frame to prevent accidental `np.asarray(huge_array)` from materializing TBs. This already exists in `TiffReaderMixin` (`arrays/_base.py:757-771`); promote it to `LazyArray` so every plugin gets it for free.

## Mixins (opt-in, composable)

Existing mixins are reorganized into `mbo_utilities/arrays/mixins/` and **all become opt-in**. None are required.

| Mixin | Provides | Lives at |
|---|---|---|
| `ReductionMixin` | `mean/min/max/std/sum/var`, `vmin`/`vmax` | `arrays/_base.py:818-1348` (move to `mixins/_reduction.py`) |
| `DimLabelsMixin` | `dims`, `slider_dims`, `dim_index`, `has_dim` | `arrays/features/_mixin.py` (already standalone) |
| `RoiFeatureMixin` | `roi`, `roi_mode`, `iter_rois`, `num_rois` | `arrays/features/_roi.py` |
| `PhaseCorrectionMixin` | `fix_phase`, `offset`, `border`, etc. | `arrays/features/_phase_correction.py` |
| `FrameRateMixin` | `fs`, `frame_interval`, `frame_rate` feature | `arrays/features/_frame_rate.py` |
| `VoxelSizeMixin` | `dx`, `dy`, `dz`, `voxel_size` feature | `arrays/features/_voxel_size.py` |
| `ImshowMixin` | `imshow()` (fastplotlib) | new — extracted from `TiffReaderMixin._imshow` |

`TiffReaderMixin` is split: its TIFF-specific bits stay in `arrays/tiff.py`; the generic bits (`astype`, `__array__`, `_imwrite`, `save`, `imshow`) become methods on `LazyArray` itself or move to small mixins. **The class name `TiffReaderMixin` will be removed** — it was a misnomer (none of its current contents are TIFF-specific).

The feature system (`ArrayFeature` base, event handlers in `arrays/features/_base.py`) is **kept as-is**. Third parties extend by subclassing `ArrayFeature` + writing a thin mixin — no registry needed. This is already documented well in the audit.

## Required attributes by ndim tier

The "core features needed to show a 2D / 3D / 4D / 5D image":

**2D (one frame, Y×X):**
Required: `shape`, `dtype`, `__getitem__`, `metadata`, `can_open`.
Display range: `vmin`/`vmax` (provided by `ReductionMixin`).

**3D (Z-stack, T-series, or 1×1×Z×Y×X with T=C=1):**
Above + nothing extra. The viewer reads `nz`/`nt` from `_shape5d()` and only builds a slider when the size > 1.

**4D (any combination of two non-spatial axes):**
Above + `dims` *recommended* (so the viewer labels the slider correctly). Without `dims`, the viewer defaults to canonical TCZYX naming — fine for most cases. Voxel size (`dz`) optional but improves Z aspect ratio.

**5D (full TCZYX):**
Above + `num_color_channels` (already auto-derived from `nc` in `LazyArray`; plugins don't need to override unless their channel count differs from `shape5d[1]`). Per-channel display: viewer picks colormaps from `metadata["channel_names"]` if present.

**Net: a plugin that implements only the 5 required attrs gets full 2D–5D viewing.** Everything else (frame rate, voxel size, ROI, phase correction) is layered via mixins for richer UIs but is never required.

## Registration & dispatch

New module: `mbo_utilities/lazy_array.py` (continued).

```python
_REGISTRY: list[type[LazyArray]] = []

def register_array_class(cls: type[LazyArray], priority: int | None = None) -> None:
    """Register a LazyArray subclass for imread() dispatch.

    If priority is given, it overrides cls.PRIORITY for this registration.
    Idempotent: re-registering the same class is a no-op.
    """

def _load_entry_point_arrays() -> None:
    """Load all classes from the 'mbo_utilities.lazy_arrays' entry-point group.
    Called lazily on first imread() — caches the result.
    """

def _dispatch(path: Path) -> type[LazyArray]:
    """Iterate registered classes in PRIORITY desc order; first can_open() wins."""
```

Built-in classes self-register at the bottom of their own module:

```python
# arrays/zarr.py — last line
register_array_class(ZarrArray)

# arrays/tiff.py
register_array_class(LBMArray)        # PRIORITY 90
register_array_class(LBMPiezoArray)   # PRIORITY 90
register_array_class(PiezoArray)      # PRIORITY 80
register_array_class(SinglePlaneArray) # PRIORITY 80
register_array_class(ScanImageArray)  # PRIORITY 70
register_array_class(TiffArray)       # PRIORITY 30 (fallback)
```

PRIORITY scheme (defaults — plugins override):
- **90**: very specific layout marker (e.g. `.corrected/` directory, LBM stack-type metadata)
- **70-80**: format + sub-pattern detection (ScanImage TIFF that isn't LBM/Piezo)
- **50**: generic format detection (default)
- **30**: last-resort fallbacks (`TiffArray` for any TIFF that didn't match a more specific class)

`pyproject.toml` declares the entry-point group:

```toml
[project.entry-points."mbo_utilities.lazy_arrays"]
zarr   = "mbo_utilities.arrays.zarr:ZarrArray"
tiff   = "mbo_utilities.arrays.tiff:TiffArray"
# ... etc
```

A fork's `pyproject.toml`:

```toml
[project.entry-points."mbo_utilities.lazy_arrays"]
myfmt = "mypkg.arrays:MyFormatArray"
```

`pipeline_registry.PipelineInfo` is **kept and complementary** — it carries declarative metadata for the CLI `mbo formats` listing, GUI dropdown labels, and marker-file documentation. It is *not* used for dispatch (dispatch is purely `can_open` + `PRIORITY`). Plugins are encouraged but not required to also `@pipeline(...)` for richer UI integration.

## The new `imread()`

`mbo_utilities/reader.py` collapses from ~500 lines to roughly:

```python
def imread(inputs, *, squeeze: bool = False, **kwargs) -> LazyArray | SqueezedView:
    if isinstance(inputs, np.ndarray):
        arr = NumpyArray(inputs, **_filter_kwargs(NumpyArray, kwargs))
        return arr.squeeze() if squeeze else arr
    if isinstance(inputs, LazyArray):
        return inputs.squeeze() if squeeze else inputs

    path = _resolve_path(inputs)  # zarr-root redirect logic stays here
    cls = _dispatch(path)
    arr = cls(path, **_filter_kwargs(cls, kwargs))
    return arr.squeeze() if squeeze else arr
```

`_resolve_path` keeps the existing zarr-inner-file → zarr-root redirect (`reader.py:178-185`).

The list-of-paths case is delegated to whichever class wins `can_open(paths[0])` — each class handles list inputs internally (most already do). Removes the cross-class file-grouping logic from `imread()`.

The isoview detection block (`reader.py:196-289`) becomes four `can_open` methods:

```python
class IsoViewCorrectedArray(LazyArray):
    PRIORITY = 90
    @classmethod
    def can_open(cls, p): return _find_isoview_corrected_root(p) is not None

class IsoviewArray(LazyArray):
    PRIORITY = 80
    @classmethod
    def can_open(cls, p):
        return p.is_dir() and (
            list(p.glob("*.stack")) and list(p.glob("*.xml"))
            or any(_TILED_FUSED_PATTERN.match(f.name) for f in p.glob("*.tif"))
        )

class IsoViewOutputArray(LazyArray):
    PRIORITY = 70
    @classmethod
    def can_open(cls, p): return bool(_find_tm_folders(p))

class ClusterPTArray(LazyArray):
    PRIORITY = 60
    @classmethod
    def can_open(cls, p): return _has_klb_in_tm_layout(p)
```

The existing helpers (`_find_isoview_corrected_root`, `_find_tm_folders`, `_has_tm_pattern`, `_TILED_*_PATTERN`) move into `arrays/isoview.py` if not already there.

## Squeeze layer (opt-in)

New module: `mbo_utilities/squeeze.py` (or in `lazy_array.py`).

```python
class SqueezedView:
    """Lightweight view of a LazyArray with size-1 dims dropped.

    Indexing on the view is translated back to the canonical 5D index on
    the underlying array. .dims is preserved (minus dropped axes).
    """
    def __init__(self, base: LazyArray):
        self._base = base
        self._kept = tuple(i for i, s in enumerate(base.shape) if s > 1)
    @property
    def shape(self): return tuple(self._base.shape[i] for i in self._kept)
    @property
    def ndim(self): return len(self._kept)
    @property
    def dtype(self): return self._base.dtype
    def __getitem__(self, key):
        full_key = _expand_squeezed_key(key, self._kept)  # fill 0 for dropped axes
        return self._base[full_key]
    @property
    def dims(self):
        if hasattr(self._base, "dims") and self._base.dims:
            return tuple(d for i, d in enumerate(self._base.dims) if i in self._kept)
        return None
```

`fastplotlib.ImageWidget` already needs singletons removed (the existing `_SqueezeSingletonDims` wrapper in the GUI) — `SqueezedView` replaces that wrapper. Internal viewer/writer code keeps working on the raw 5D array (no squeeze); user-facing notebook ergonomics get `arr.squeeze()` or `imread(path, squeeze=True)`.

## Downstream coordination: LBM-Suite2p-Python

LBM-Suite2p-Python consumes mbo_utilities arrays directly and breaks on the always-5D
contract once the `shape5d` alias is removed. It is updated **eagerly in the v4 rollout**
(not deferred to the grace period), and its array detection switches from a hardcoded
class-name list to `isinstance(obj, LazyArray)`.

Prerequisite in mbo_utilities: **export `LazyArray` at the top level** so the isinstance
check has a stable import. Add `"LazyArray"` to `mbo_utilities/__init__.py` `__all__` and a
branch in its lazy `__getattr__` (`from .lazy_array import LazyArray`). Keep it import-light
— `lazy_array.py` must not pull heavy deps at import time.

Integration points (suite2p repo):

| Location | Current | v4 work |
|---|---|---|
| `utils.py:7` `_LAZY_ARRAY_TYPES` + `_is_lazy_array` | tuple of class-name strings (stale: missing `IsoView*`, lists possibly-dead `MBOTiffArray`/`MboRawArray`) | replace body with `isinstance(obj, mbo_utilities.LazyArray)`; keep a string-name fallback only for objects from a mbo version too old to expose `LazyArray` |
| `utils.py:_get_num_planes` | prefers `arr.nz`, then 5D `shape[2]`, with 4D/3D fallbacks | works as-is under 5D; the 4D/3D fallbacks become dead but harmless — leave or trim |
| `cellpose.py:55-73` | `ndim == 5 / 4 / 3` branching for plane extraction | only the 5D branch is reachable; keep 4D/3D as explicit legacy-input guards |
| `grid_search.py:251-257` | `ndim == 5 / 4` branching for `plane_Ly/plane_Lx` | same — 5D path is canonical; `shape[-2:]` already covers all ranks |
| `run_lsp.py:1384` | `tuple(input_arr.shape5d)` | → `tuple(input_arr.shape)` |
| `run_lsp.py:2738` | `tuple(file.shape5d)` | → `tuple(file.shape)` |
| `run_lsp.py:2876-2877` | `if hasattr(chan2_data, "shape5d"): ops["nframes_chan2"] = chan2_data.shape5d[0]` | → `chan2_data.shape[0]` (now always real T) |
| `run_lsp.py:2609, 2639-2641` | `input_arr.shape[0]`, `shape[-2:]` | no change — already 5D-correct |

The `_ChannelView` wrapper (mbo_utilities `gui/tasks.py`, wrapping 5D TCZYX as 4D TZYX for a
single channel, registered in suite2p's lazy-array set) and the `run_plane` per-channel
extraction path must be re-verified against current code: under always-5D the wrapper's input
is guaranteed 5D, so its 4D-output contract is unchanged but its assumptions should be
re-checked once `_LAZY_ARRAY_TYPES` is replaced by isinstance (the wrapper must still pass the
isinstance test, i.e. subclass `LazyArray` or be detected another way).

Release ordering: ship mbo_utilities v4 first (with `shape5d` still aliased), then the suite2p
update pinning `mbo_utilities >= 4.0`. Because the alias survives one minor, suite2p does not
hard-break the instant v4 lands; the eager update removes the `shape5d` references before the
alias is dropped in the following minor.

## Phased rollout

Each phase lands as its own PR; tests stay green throughout.

1. **Phase 1 — base + registry, no behavior change.**
   Land `lazy_array.py` (`LazyArray`, `register_array_class`, entry-point loader) alongside the existing system. Existing arrays inherit `Shape5DMixin` *and* `LazyArray` (LazyArray is empty of conflicts). Add no-op `can_open` defaults. Tests still pass — old `imread` still uses if/elif.

2. **Phase 2 — convert one format at a time.**
   Suggested order (least disruptive first): `NumpyArray` → `H5Array` → `BinArray` → `ZarrArray` → `Suite2pArray` → `TiffArray` → ScanImage subclasses → IsoView family. For each: lock `.shape` to 5D, add `can_open`, register. Update its tests (5D shape assertions).

3. **Phase 3 — switch dispatch.**
   Rewrite `reader.py` to use `_dispatch()`. Delete the if/elif chain. Run full test suite (139 pass, 37 skipped today — target same).

4. **Phase 4 — squeeze layer + GUI integration.**
   Land `SqueezedView`. Replace `_SqueezeSingletonDims` wrapper in GUI. Add `imread(squeeze=True)` kwarg.

5. **Phase 5 — docs + entry points.**
   `pyproject.toml` declares all built-ins via entry points. Write `MIGRATION.md` and `docs/forking.md`. Add a cookiecutter or example fork (a tiny plugin reading a contrived format) under `examples/forking/`.

6. **Phase 6 — clean up memories.**
   Update the three memory files identified above. Tag v4.0.0.
